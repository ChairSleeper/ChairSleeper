I'm someone who loves understanding how things *really* work — all the way down to the metal.

Have you ever tried printing `Hello, World` in C **without `printf`**…  
or booting a computer **without GRUB**, using a bootloader you wrote yourself in Assembly?

Yeah. That's my kind of Saturday.

I'm a B.Tech student obsessed with the internals of computers — operating systems, memory management, CPU scheduling, and how programs behave at the lowest level. On the other side, I fine-tune and quantize large language models, because understanding *how AI thinks* at the systems level is just as interesting as understanding how CPUs do.

I write code because I'm curious, not because I have to.  
And every day, I try to push one layer deeper into how computers actually work.

## 🚀 Featured Project

### [AxiomX-OS](https://github.com/ChairSleeper/AxiomX-OS) — x86_64 Operating System from Scratch

> Built entirely without GRUB. No shortcuts. No training wheels.

A fully custom x86_64 operating system featuring:
- 🔧 **4-stage bootloader** — Real mode → Protected mode → Long mode → 64-bit kernel, written in pure Assembly & freestanding C
- 🧠 **Physical & Virtual Memory Manager** with paging
- ⚡ **Round-robin scheduler** with full context switching
- 🖥️ **Interactive shell** with 24 built-in commands
- ⏱️ **IDT + PIT** interrupt handling
